# Specs

Esta carpeta guarda specs livianas para cambios que merecen mas claridad antes de implementarse.

## Cuando crear una spec

- cuando una fase cambia estructura o arquitectura
- cuando una fase toca varias secciones de la home
- cuando una fase introduce i18n, deploy o integraciones
- cuando una fase necesita criterios de aceptacion claros

## Estructura sugerida

- objetivo
- alcance
- tareas
- restricciones
- criterios de aceptacion
- validacion

## Regla

- una fase relevante puede apuntar a una spec concreta dentro de esta carpeta
- no usar specs para cambios triviales
